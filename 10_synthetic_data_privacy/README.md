# 10 — Synthetic Data and Privacy

This module introduces synthetic healthcare data generation and privacy-aware sharing.

## Why This Matters

Healthcare data often contain protected health information (PHI). Public repositories should never include identifiable patient records.

Synthetic data can help support:

- reproducible tutorials
- safe public examples
- tool demonstrations
- education
- pipeline testing
- dashboard prototyping

## Topics Covered

- PHI risk awareness
- synthetic patient data generation
- privacy-preserving examples
- safe public sharing
- responsible repository practices

## Files

| File | Purpose |
|---|---|
| generate_synthetic_health_data.py | Creates synthetic patient and encounter data |
| privacy_notes.md | Privacy and PHI guidance |
| synthetic_data_dictionary.md | Variable definitions |
| data/ | Generated synthetic data |
| outputs/ | Summary outputs |

## Important Note

Synthetic data are for education and demonstration. They should not be treated as real clinical data.
