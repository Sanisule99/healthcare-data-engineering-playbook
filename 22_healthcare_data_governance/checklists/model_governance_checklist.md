# Model Governance Checklist

Use this checklist before presenting or deploying a healthcare ML model.

## Model Documentation

- [ ] Model purpose documented
- [ ] Target outcome defined
- [ ] Input features listed
- [ ] Training data described
- [ ] Validation data described
- [ ] Intended users identified

## Performance

- [ ] Accuracy or primary metric reported
- [ ] Calibration assessed
- [ ] Error analysis completed
- [ ] Subgroup performance reviewed
- [ ] External validation considered

## Explainability

- [ ] Feature importance reviewed
- [ ] Clinical plausibility assessed
- [ ] Model limitations documented
- [ ] Human review workflow described

## Safety

- [ ] Not used as sole clinical decision-maker
- [ ] Bias risks documented
- [ ] Monitoring plan considered
- [ ] Data drift considered
- [ ] Responsible use statement included
